Just closed this session

The two root-cause fixes requested, plus the guardrails landed earlier in the session.

✓ Lease fencing on terminal writes queue-service.ts · scheduler.ts

WasupdateStatus fenced terminal writes with lease_token == null || == token. After a reclaim nulled the token, a zombie/late daemon write still matched the == null branch → spurious failed stamped over an entry another owner (or a fresh pending) controlled. finalizePipeline and the detached-poll takeover/error handlers wrote terminal status with no token at all.
NowDedicated complete(id, token) / fail(id, msg, token): read the doc, require an exact lease-token match (no null escape), reject already-terminal (idempotent), then write with if_seq_no/if_primary_term optimistic concurrency — retrying on a 409 (a heartbeat racing the write) while re-checking the lease each attempt. The token is threaded to every terminal-write site, including the detached Buildkite poll (leases are no longer dropped for deferred entries so they keep heartbeating).
ImpactA daemon whose lease was reclaimed/adopted can never clobber the live owner. Kills the spurious-failed class. Also fixed a latent bug: a fenced-out failure no longer triggers a duplicate re-enqueue.
Verified7 new complete/fail fencing tests + reclaim excludeIds test; 1027 unit tests green; tsc + lint clean; build OK.

✓ Periodic stuck-entry reclaimer scheduler.ts · queue-service.ts

WasreclaimStaleEntries() ran once on startup (cli.ts). A run stuck in deploying/benchmarking while the daemon kept running required a manual curl reset to pending.
NowThe scheduler runs the reclaimer every poll (throttled to once per heartbeat TTL, 120s) as a backstop. It resets entries stale past the TTL back to pending and never touches entries this daemon owns (a new excludeIds guard covers active leases + in-process + deferred cleanups; owned entries are also kept fresh by the heartbeat).
ImpactNo more manual resets. A crashed/abandoned run self-heals to pending and is retried automatically.

✓ Guardrails landed earlier this session

  • Daily cost cap — costCaps.maxModelsPerDay: 15 gates new dequeues once the daily terminal-model ceiling is hit; never stops the VM, never touches in-flight/resume runs. Fails open on a count-query error.
  • Failure classifier + auto-retry — transient-infra / resource-fit / model-arch / unknown; retriable classes auto-re-enqueue within budget (maxInfraRetries: 2, maxResourceFitRetries: 1), the rest quarantine.
  • VM disk-GC (LRU) — pre-deploy free-space check LRU-evicts the oldest model weight sets from the vllm-hf-cache Docker volume, protecting the deploy target.
  • Discovery scheduler ON — discoveryScheduler.enabled: true, polls HF daily, auto-queues hardware-fit candidates gated behind the cost cap.

Closed — P0 (were blocking true unattended operation)

✓ Daemon supervision / auto-restart deploy/*.plist.template · install-launchd.sh · start-local.sh

WasThe daemon ran as a foreground process; a crash or reboot left nothing to restart it.
NowA committed LaunchAgent template (deploy/com.elastic-llm-benchmarker.plist.template) with KeepAlive+RunAtLoad+30s ThrottleInterval, rendered into ~/Library/LaunchAgents by scripts/install-launchd.sh (real paths never touch the repo). Recovery is safe: startup reclaim + resume/adopt pick up in-flight entries, so KeepAlive never double-processes a model. Log-rotation solved: start-local.sh tail-truncates the daemon log (keep last ~5MB above 50MB) on each launchd respawn.
OperatorOne command: ./scripts/install-launchd.sh (loads it) or --render to preview. The in-process health digest surfaces a stale-lease alert; pair with an external cron for defense-in-depth.

✓ CI-eval guard checked a stale tunnel URL ci-eval-infrastructure-guard.ts

WasThe guard captured publicEndpointUrl once at construction and health-checked the dead URL forever after a tunnel rotated, "recovering" onto a different live tunnel every loop — one refactor from a spurious takeover/abort of a healthy eval.
NowThe guard tracks a mutable currentPublicUrl; on reconnect it adopts the new tunnel URL and logs that the Buildkite-injected URL is stale, breaking the flap loop. Regression test asserts the guard adopts the rotated hostname and does not loop.

✓ VM cost accrual & idle detection maintenance-scheduler.ts · benchmarker-vm-cost

ConstraintPermanent: the GPU VM is ephemeral — stopping it loses it. The pipeline never stops the VM.
NowThe daily maintenance tick emits a cost/utilization snapshot (utilization_ratio = benchmark-hours / wall-hours, estimated_cost_usd = vmHourlyCostUsd × wall-hours) to the new benchmarker-vm-cost index and flags low_utilization below maintenance.lowUtilizationThreshold (default 20%). No auto-stop — the lever is raising maxModelsPerDay / widening discovery.

Closed — P1 (reliability / observability)

✓ Health-digest alerting when the pipeline wedges slack-notifier.ts · maintenance-scheduler.ts

WasSlack fired on per-model verdicts only; nothing paged when the pipeline stuck (cost cap engaged, DLQ growing, discovery idle, golden forwarding failing, daemon lease stale).
NowSlackNotifier.notifyHealthDigest() posts a daily digest from the maintenance tick with a breach-aware header (:rotating_light: + alert count). Signals: daemon-lease age, queue depth, throughput (idle detection), VM utilization/burn, cost-cap state, errors logged (incl. golden-forward category), and DLQ re-tries. Threshold breaches are logged and headlined.

✓ Quarantine / DLQ periodic re-try queue-service.ts · maintenance-scheduler.ts

WasA retry-budget-exhausted model sat failed forever; no periodic re-evaluation (a transient-infra failure may become runnable after a vLLM image bump) and no category surface.
NowThe maintenance tick re-classifies failed entries older than maintenance.dlqRetryAfterDays (default 7) via the failure classifier and re-enqueues only the retriable ones back to pending (requeueFailedEntries), bounded by dlqMaxRequeuePerSweep and gated by the daily cost cap. Quarantine counts by status surface in the digest.

✓ Discovery de-dup & freshness discovery-scheduler.ts · queue-service.ts

WasDiscovery could re-surface models already benchmarked or quarantined, spending cost-cap budget on repeats. (Latest-per-family supersession already existed.)
NowBefore auto-queue, DiscoveryScheduler queries QueueService.findRecentTerminalModelIds() and skips any model terminal (completed or quarantined) within discoveryScheduler.skipRecentlyBenchmarkedDays (default 30). Fails open on an ES error so discovery is never blocked. Complements the existing markSupersededByFamily latest-per-family preference.

✓ Golden-forwarding health monitored folded into the health digest

WasForwarding 429s/503s landed in error indices but nothing surfaced them — silent data loss to the shared cluster.
NowThe health digest counts recent benchmarker-errors and a dedicated golden-forward category via countErrorsSince(), alerting when any golden-forward error is logged. Golden stays write-only.

✓ Secret-hygiene guard generalized pre-commit-check-credentials.sh

WasThe pre-commit hook only caught one hardcoded VM IP and one home dir — a different real IP, token type, or key path would slip into a tracked file.
NowThe hook rejects any real routable IPv4 (awk range-filter excludes loopback/RFC1918/link-local/TEST-NET/placeholder), any macOS/Linux home-dir path or SSH private-key path, and a broader token set (GitHub, HF, Slack, AWS, GCP, PEM blocks), plus the .env*/config/local.json files themselves. Rotation policy note stands: rotate anything that reached a shared branch.
